# v1.3.0 - Src Layout Reorg

## Summary

The Python implementation now lives under `src/research_pipeline/` while the old root-level command and import surfaces remain as compatibility wrappers.

## What Moved

- `tools/*` implementation -> `src/research_pipeline/tools/*`
- `tools/evals/*` implementation -> `src/research_pipeline/tools/evals/*`
- `gates/*.py` implementation -> `src/research_pipeline/gates/*.py`
- `scripts/*.py` implementation -> `src/research_pipeline/cli/*.py`

## What Stayed At Root

- `scripts/*.py` wrappers for existing user commands
- `tools/*.py` and `tools/evals/*.py` shims for `python -m tools.*`
- `gates/*.py` shims for legacy imports such as `from gates.output_gates import ...`
- `pipeline.json`, `schemas/`, `.claude/`, `.hcom/`, `inputs/`, `outputs/`, `evals/`, and `observability/`

## Path Safety

All implementation modules should import shared paths from `research_pipeline.paths` instead of deriving repo root with `Path(__file__).resolve().parents[...]`.

## Validation

- `python -m py_compile ...` passed for wrappers and core modules.
- `python scripts\validate_specs.py` passed with `ALL VALIDATIONS PASSED`.
- `python scripts\smoke_v2.py` passed with `ALL V2 SMOKE CHECKS PASSED`.

`python scripts\status.py` could not be verified in the Codex sandbox because process spawning was blocked with `windows sandbox: spawn setup refresh`; this appears sandbox-level, not a Python import failure.
