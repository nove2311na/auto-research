# v1.5.0 - Swarm Security, Gates Validation, and Observability Dashboard

## Summary

Implemented comprehensive validation logic for implementation and release gates, enhanced input validation with URL probing and file size checks, expanded security static analysis (path traversal, credentials, and size checks), created security red-team test cases, automated learnings update on critic failures, designed an observability render dashboard, and established a GitHub Actions CI workflow.

## Features & Changes

### Swarm Gates & Safety
- **Implementation Gate (`implementation_gates.py`)**: Implemented validation check for artifact version formats (`v\d+`), stage directory existence, and presence of parent stage winner in the manifest.
- **Release Gate (`release_gates.py`)**: Implemented verification that all 6 stages have successfully selected winners, the pipeline is finalized with `completed_at`, and `05_format/v1.md` exists and is non-empty.
- **Input Probing (`input_gates.py`)**: Extended input completeness checks with URL regex matching, responsive HTTP probes (HEAD/GET requests with redirects and timeouts), and 5MB maximum file size constraints.
- **Security Enhancements (`security_gates.py`)**: Expanded secret patterns scanner with regex checks (AWS credentials, GitHub PAT, Slack webhooks, generic API keys), enforced 2MB maximum file size limits to prevent DoS, and added absolute path traversal blocks (`../` / `..\\`).
- **Inter-run Deduplication (`run_pipeline.py`)**: Added checks against `is_done(input_id)` to skip already successfully completed research runs unless a `--force` flag is provided to override.

### Observability & Telemetry
- **Swarm Execution Dashboard (`render_dashboard.py`)**: Developed an observability dashboard CLI tool and library aggregating runs, gates, tool calls, and memory reads/writes from `observability/traces/` along with performance data from `trace.jsonl` to output JSON or print ASCII summaries.
- **Aggregate Metrics**: Integrated automatic appending of run stats (token count, tool calls, duration, average score, success status) to a unified log `observability/aggregate_metrics.jsonl` on finalizing.
- **URL Caching & Retries (`fetch_input.py`)**: Added global JSON-based response caching under `outputs/.cache/` and 3-attempt exponential backoff retry for url downloads.

### Ingestion, Synthesis & Formatting Fallbacks
- **Semantic Chunking (`fetch_input.py`, `ingestor.md`)**: Replaced raw 1MB truncation with keyword-relevance based semantic chunking (dividing into 40KB segments and sorting by keyword density against the topic) when input length exceeds 1MB.
- **Self-Rebuttal Metadata (`artifact_io.py`, `synthesizer.md`)**: Configured `build_meta` and the Synthesizer agent to write its internal self-rebuttal reviews directly into `v1.meta.json` using `self_rebuttal_passed` and `self_rebuttal_notes`.
- **Formatter Fallbacks (`formatter.md`)**: Programmed default fallback arrays `[]` for optional elements like `diagrams` and `theses` in case upstream stages fail to produce them, preventing formatting halt.

### Quality & CI/CD
- **Critic Failures Loop**: Configured `validator.py` and `.claude/agents/critic.md` to automatically log quality issues and validation failure patterns to `learnings.md` whenever a validation score drops below 0.5.
- **Security Red-Team**: Formulated security adversarial cases (`prompt_injection_01.txt`, `path_traversal_01.txt`, `oversized_01.txt`, and `cases.json` index) under `evals/security-redteam/`.
- **GitHub Actions CI Workflow**: Added `.github/workflows/smoke.yml` to automatically install `uv` package manager and dependencies, run pipeline smoke tests, and execute evals on every commit.

## Verification

- **Smoke test (`smoke_v2.py`)**: Passed successfully with `aggregate_metrics.jsonl` and `learnings.md` being updated.
- **Validation Gates**: Verified that version checks, directory checks, traversal blocks, size limits, and secret scanners fail and pass appropriately with test scripts.
- **Observability Dashboard**: Aggregated trace runs successfully and output a complete ASCII summary.
