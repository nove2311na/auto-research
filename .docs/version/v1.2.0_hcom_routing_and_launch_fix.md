## 1. Executive Summary

This release resolves critical communication and routing bottlenecks in the multi-agent `hcom` swarm. Previously, launching a batch of 7 agents under a shared `research-pipeline` tag generated random, non-deterministic CVCV names (like `research-pipeline-zulu`). These names did not match the hardcoded target addresses (like `@research-pipeline-claude-1`), causing all message handoffs and kickoff calls to fail. 

Additionally, trying to attach inline bundles (via `--title`) during the initial kickoff failed because the `hcom v0.6.13` validator strictly requires non-empty `transcript` and `events` references, which do not exist during startup.

This version sequentializes the swarm launch, assigns designated prefix-routable tags, and refactors all kickoff scripts to use plain hcom messaging, resolving these communication bugs completely.

## 2. Key Objectives Accomplished

1. **Sequential Agent Tagging (`scripts/launch.py`)**: Spawns 8 agents one by one with sequential `HCOM_TAG` environment variables (`research-pipeline-claude-1` through `8`). This enables precise prefix-match routing in `hcom` so `@research-pipeline-claude-N` successfully maps to its designated agent.
2. **Default Swarm Size Boost**: Increased default agent count from 7 to 8 to support the full 8-agent swarm (1 orchestrator + 7 workers) mandated by `AGENTS.md`.
3. **Plain Text Kickoffs (`scripts/kickoff.py` & `scripts/run_pipeline.py`)**: Overhauled `hcom send` wrappers to send plain hcom text messages instead of attempting inline bundle creation. This avoids validation crashes for missing transcripts/events on startup.
4. **Valid Specs Execution**: Verified that all agent and skill specifications pass validation tests seamlessly.

## 4. SWARM TROUBLESHOOTING & RECOMMENDATIONS

### 4.1. MiniMax/Anthropic 401 Unauthorized Error
During background diagnostic scans of the `research-pipeline-zulu` agent, we identified a persistent authentication failure:
```json
401 {"type":"error","error":{"type":"authentication_error","message":"login fail: Please carry the API secret key in the 'X-Api-Key' field of the request header"}
```
* **Implication**: The background agents crashed immediately upon launching because their `ANTHROPIC_AUTH_TOKEN` was not loaded or was invalid for the custom MiniMax endpoint (`https://api.minimax.io/anthropic`).
* **Fix**: Ensure your `.env` file or local command prompt environment has a valid token set before launching:
  ```cmd
  set ANTHROPIC_AUTH_TOKEN=your_real_key_here
  ```
