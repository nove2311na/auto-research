# v1.4.0 - Swarm Optimization, Budgets, and Evals

## Summary

Implemented loop budgets, parallel extraction capabilities, dynamic stage-trigger skill configurations, trace metrics collection, automated evaluations, semantic lookup over learnings, and a telemetry cost dashboard.

## Features & Changes

### Swarm & Loop Controls
- **Loop Budgets**: Added limits in `pipeline.json` and verification rules in `.claude/agents/orchestrator.md` checking retries, wall time, tool calls, and context tokens.
- **Parallel Extraction**: Enabled `"execution_mode": "parallel"` for the extract stage, sending concurrently routed options (A, B, C) messages to distinct agent tags.
- **Depth-Aware Search**: Dynamically configures iteration count and search source limits in `researcher.md` based on `--depth` flag inputs.
- **Progressive Skill Loading**: Added support for dynamic, on-demand loading of stage skill dependencies (`skill_triggers`).

### Observability & Telemetry
- **Trace Logs**: Stage metrics (duration, tokens, tools used) are appended to `outputs/<input_id>/trace.jsonl` upon completion.
- **Swarm Cost Dashboard**: Integrated telemetry analytics directly into `status.py`, displaying total tokens, tool calls, and estimated run cost in USD.

### Memory & Quality
- **Auto-Learnings**: Runs append summaries and score stats automatically to `learnings.md` when finalizing.
- **Semantic Memory**: Added a retrieval script `memory_retriever.py` permitting agents to search historical learnings by keyword.
- **Evals Framework**: Built `eval_pipeline.py` checking completeness, theme-to-fact coherence, and prompt injection attempts.
