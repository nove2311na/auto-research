# Workspace Artifact Schemas

Runtime artifacts live under `workspace/` and are generated by the PM, architect, operator, and gatekeeper. The schemas in `agentic/schemas/` define the expected shape.

## Core Files

| File | Producer | Purpose | Schema |
|---|---|---|---|
| `workspace/meta.json` | workspace steward | Project identity, source URL, runtime, initialization timestamp. | `agentic/schemas/meta.schema.json` |
| `workspace/page_structure.json` | architect | Page hierarchy, Client-First layers, target Webflow nodes. | `agentic/schemas/page-structure.schema.json` |
| `workspace/blueprints/*.json` | architect | Approved build plan for one page or component. | `agentic/schemas/blueprint.schema.json` |
| `workspace/state.json` | all agents by role | Phase transitions, ReAct traces, approvals, reflection reviews. | `agentic/schemas/state.schema.json` |
| `workspace/error-logs.json` | QA and PM | Issues, fix owners, resolution status. | `agentic/schemas/error-log.schema.json` |
| `workspace/qa-report.json` | architect or gatekeeper | Visual QA result and approval/fix verdict. | `agentic/schemas/qa-report.schema.json` |

## ReAct Action Entry

```json
{
  "timestamp": "2026-06-02T00:00:00Z",
  "agent": "figma-webflow-operator",
  "phase": "phase_2_webflow_build",
  "type": "webflow_action",
  "reason": "Hero section needs a semantic section wrapper before content blocks.",
  "action": "create element section with class home-hero_section",
  "observation": "Webflow returned element id el_hero_section and accepted class assignment.",
  "next_decision": "create container and content wrapper",
  "artifact": "workspace/state.json",
  "resolution": "complete"
}
```

## Approval Entry

```json
{
  "timestamp": "2026-06-02T00:00:00Z",
  "agent": "pm",
  "phase": "phase_1_blueprint",
  "type": "approval",
  "approver": "user",
  "artifact": "workspace/blueprints/home.json",
  "message": "Approved blueprint home.json for phase_2_webflow_build",
  "resolution": "complete"
}
```

## Versioning

Do not overwrite artifacts. Create a new blueprint filename or append a new state entry when a phase repeats.
