# Skill 05 — Format

## Identity
- Stage id: `05_format`
- Owning agent: `formatter`
- Schema: `schemas/05_format.json` (for JSON variant); Markdown is human-readable
- Output format: `json+md` (per `pipeline.json:57`)
- `max_options`: 1, `max_retries`: 3

## Input schema
| Field | Type | Required | Source |
|---|---|---|---|
| `input_id` | string (8 hex) | yes | hcom message from `@orch` |
| `04_synthesize/v1.json` | file path | yes (main input) | upstream artifact |
| `02_extract/v1.json` | file path | yes (for entities + facts) | upstream artifact |
| `03_analyze/v1.json` | file path | yes (for themes/gaps/contradictions) | upstream artifact |

## Process (see `skill.json#process` for the structured form)

1. **(deterministic)** Read 3 upstream artifacts.
2. **(LLM + deterministic)** Build `v1.json` per `schemas/05_format.json` using the field-mapping below.
3. **(deterministic)** Write `v1.json` via `write_artifact`.
4. **(LLM + deterministic)** Render `v1.md` using the Markdown template (below). **The formatter adds the ` ```mermaid ` wrapper.**
5. **(deterministic)** Write `v1.md` via `write_artifact` with `ext="md"`.
6. **(LLM + deterministic - Academic Visual Assets)** Scan preceding stages for quantitative, structural, or statistical data. If present:
   - Generate `chart.py`: A publication-ready Python script using `matplotlib`/`seaborn` with academic styling (clean grid, professional fonts, HSL-tailored colors) that generates a clean PDF/PNG chart when run.
   - Generate `table.tex`: A clean LaTeX `table` or `tabular` environment using standard `booktabs` package rules (no vertical lines, clear `\toprule`, `\midrule`, `\bottomrule`).
   - Write them using `write_artifact` with `ext="py"` and `ext="tex"` respectively in the stage directory.
7. **(deterministic)** Write `v1.meta.json` (validation: pending).
8. **(deterministic)** Ping `@critic`.
9. **(deterministic, ONLY if critic passes)** Call `tools.manifest.finalize(input_id)`. Report `done` to `@orch`.

## Rigorous Rules

### 1. Anti-AI Writing Rules (Viết Súc Tích - Phi AI)
- **Cắt bỏ hoàn toàn các từ đệm, mở đầu rườm rà**: Tránh dùng "In order to...", "It is important to note...", "Due to the fact that...". Hãy đi trực tiếp vào chủ ngữ và hành động.
- **Danh sách từ cấm (AI Vocabulary)**: Tuyệt đối không sử dụng các từ: "delve", "testament", "crucial", "landscape", "moreover", "furthermore", "tapestry", "hub", "enrich", "empower". Thay vào đó hãy dùng các động từ/tính từ đơn giản như "is/are", "shows", "adds", "has", "proves".
- **Nhịp điệu tự nhiên**: Đa dạng hóa cấu trúc và độ dài của các câu. Tránh viết 3 câu liên tiếp có cùng số lượng từ hay cùng cấu trúc ngữ pháp.
- **Academic Trust**: Trình bày thông tin khách quan, trực diện. Tránh dùng các từ làm nhẹ bớt mức độ khẳng định một cách vô nghĩa ("could potentially be argued that...").

### 2. Obsidian KB Network Linking Rules (Liên Kết Tri Thức)
- Tự động nhận diện các thực thể/khái niệm khoa học quan trọng nhất trong báo cáo và bọc chúng bằng liên kết hai chiều kiểu wikilinks: `[[Tên Khái Niệm]]` hoặc `[[Tên Tổ Chức]]` (ví dụ: `[[Neural Networks]]`, `[[OpenAI]]`).
- Đảm bảo YAML frontmatter đầy đủ, chính xác, định dạng thẻ tags hợp lệ để người dùng có thể kết nối trực tiếp vào đồ thị Obsidian Vault của họ.

## Self-check (see `skill.json#self_check` for the full numbered list)
- All required fields in `v1.json`
- Every field traces to one of: `04_synthesize`, `02_extract`, `03_analyze`, `01_ingest` (no invented content)
- `entities[]` deduplicated by name
- `facts[]` 10-20 entries (filtered by `confidence` if more)
- `insights[]` is `string[]` (not array of objects)
- `diagrams[]` matches `04_synthesize.diagrams[]` 1:1
- `theses[]` matches `04_synthesize.theses[]` 1:1
- `references[]` includes `01_ingest.source_ref` + all cited URLs
- Mermaid wrappers in `v1.md` (the `v1.json` `code` field is unwrapped)
- Title H1 = `<title or first 80 chars of summary>`
- JSON ↔ Markdown agree; no new facts in Markdown
- After critic pass: `tools.manifest.finalize(input_id)` called
- On critic fail: do NOT call `finalize`

## Validation
- Schema + completeness + LLM-judge (on `v1.json`); Markdown is human-checked
- Common failure: `references` empty → add `01_ingest.source_ref` and cited URLs
- Common failure: Mermaid missing wrapper in `v1.md` → wrap
